Performance

Table: Performance Graphs File Locations

Components

The Performance application includes two components:

Performance Configuration – Configure graphing options for checks that have performance data generation enabled.

Performance Viewer – View a group RRD graphs created by the checks.

Applications

The application consists of two Perl scripts located in:

/usr/local/groundwork/core/performance/cgi-bin/performance

PerfConfigAdmin.pl – Performance Configuration UI that stores data in Monarch's performanceconfig table.

perfchart.cgi – Charting UI that creates views which are stored in:

/usr/local/groundwork/core/performance/performance_views

Application viewer (Framework Package)

The JBoss Portal page is located at: Administration>Portal Management>groundwork-monitor portal>config page>Performance page

Reports

GroundWork Reports

[Enterprise only]

Table: GroundWork Reports File Locations

Components

The groundwork report server  includes two components:

Foundation-reportserver - An application to manage BIRT reports located in: /usr/local/groundwork/foundation/container/webapps/foundation-reportserver.war

birtviewer - An Eclipse application to view reports generated with Eclipse BIRT Report Designer located in: /usr/local/groundwork/foundation/container/webapps/birtviewer.war

Configuration Files

GroundWork Reports configuration files are located in:

/usr/local/groundwork/config/gwreportserver.properties

Starting/Stopping Report Server Application

The report server is part of the Foundation deployment. More information about how to run can be found in Configuring Foundation.

Application Viewer (Framework Package)

The JBoss Portal page is located at: Administration>Portal Management>groundwork-monitor portal>reports page

Insight Reports

Table: Insight Reports File Locations

Application Viewer (Framework Package)

The JBoss Portal page is located at: Administration>Portal Management>groundwork-monitor portal>reports page

CGI Programs

CGI programs that create and present the report to a browser run under the Guava portal framework and are located in the /usr/local/groundwork/core/reports/cgi-bin/reports directory.

Data Load Utilities

The data load utilities are located in the /usr/local/groundwork/reports/utils directory. These scripts load the dashboard database by reading the Nagios log files and are executed by cron under user nagios. The log messages generated by these scripts are in the directories:

/usr/local/groundwork/reports/utils/log

/usr/local/groundwork/nagios/var

Configuration File

The reporting configuration file that controls the reporting options is:  /usr/local/groundwork/core/reports/etc/gwir.cfg

Status

Table: Status File Locations

Package

The Status package is located in /usr/local/groundwork/core/guava/htdocs/guava/packages/sv2 directory. This directory contains all required PHP, Javascript, and image files for Status to function.

Status Guava Application Class

The /usr/local/groundwork/core/guava/htdocs/guava/packages/sv2/apps directory contains the main Status Guava application class, SV2Application. This is the principal class that handles the UI for Status.

Status Viewer System Module Class

The /usr/local/groundwork/core/guava/htdocs/guava/packages/sv2/sysmodules directory contains the Status system module class which handles mostly messaging and I/O for the Status application .

Images

The  /usr/local/groundwork/core/guava/htdocs/guava/packages/sv2/images directory contains the images used in the Status application.

Components

The  /usr/local/groundwork/core/guava/htdocs/guava/packages/sv2/support directory contains the classes that define components of the Status application such as the HostView and Overview.
