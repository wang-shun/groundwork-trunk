Configuring Event Console

Configuring Internationalization

The Event Console application supports Internationalization (i18n). The standard supported languages include English, French and others, and are configurable using new message bundles. Note that only the labels are localized and not the data.

Configuring the Event Console Locale from a Command Line

In this example we will configure the Event Console to the French locale. The ISO 639-1 code for the French language is fr. Refer to this link; List of ISO 639-1 Codes for a complete listing of all the ISO 639 codes.

  1. Open the file: /usr/local/groundwork/config/console.properties

  2. Change the locale property code

from English (en)

#Internationalization

locale=en

to French (fr)

#Internationalization

locale=fr

Configuring the Event Console Locale from the UI

Internationalization can also be configured from within GroundWork Monitor.

  1. Select Administration>Foundation>Manage Configuration.

  2. Next select the configuration file console.properties, as displayed in the image below.

  3. Scroll down the list and you will see the property locale. Enter the desired value. For example: fr for French.

  4. Select Add, and then Save. The new property value will be added to the console.properties file.

Figure: Setting the Locale in GroundWork Monitor Administration

Adding a New Message Bundle

In this example we will add a new message bundle for French.

  1. Open the file: /usr/local/groundwork/config/console.properties

  1. Change the current locale property code to: jp for Japanese.

#Internationalization

locale=jp

  1. Next you will need to translate all of the messages into a new file by copying the file: messages_en.properties (use current locale property) to a new file named: messages_jp.properties

  2. Then, restart gwservices with the following command:

/usr/local/groundwork/ctlscript.sh restart gwservices
