Configuring Event Console

Masking Dynamic Columns in the Event Console

In the Event Console, the dynamic properties for any application type can be masked/made invisible, thus excluded from the Event Console view. To do this the invisible.dynamic.columns property is added in the console.properties file, the entry is case sensitive and multiple values are separated by commas. The value needs to match to the PropertyType database table.

  • See Appendix A to view the entire console.properties file.

  • To view the PropertyType database table you can use the following:

postgresql>select PropertyTypeID, Name, Description from PropertyType

Masking a Dynamic Column from a Command Line

  1. Open the file: /usr/local/groundwork/config/console.properties

  2. Enter one or more properties to be excluded. For example;

To hide the column updatedBy in the UI, change the property

from:

#invisible.dynamic.columns=updatedBy

to:

invisible.dynamic.columns=updatedBy

or to hide the columns updatedBy and ipaddress in the UI, change the property

from:

#invisible.dynamic.columns=updatedBy

to:

invisible.dynamic.columns=updatedBy,ipaddress

If this property is commented out or empty, all dynamic properties are shown:

#invisible.dynamic.columns=updatedBy

invisible.dynamic.columns=

Masking a Dynamic Column from the UI

Dynamic Columns can also be configured from within GroundWork Monitor.

  1. Select Administration>Foundation>Manage Configuration.

  2. Next select the configuration file console.properties, as displayed in the image below.

  3. If the Property Name invisible.dynamic.columns is not listed, then in the New Property box type invisible.dynamic.columns and enter the property value(s). For example: invisible.dynamic.columns and udatedBy,ipaddress

  4. Select Add, and then Save. The new property will be added to the properties file.
