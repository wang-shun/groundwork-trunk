Importing Service Profiles and services

Installed and Distributed Profiles

Profiles contain both pre-defined and user-definable monitoring parameter settings and can be imported into the Configuration tool for use in designing and managing Hosts and Services. Profiles can also be exported making Profiles portable and enabling the sharing of files.

GroundWork Monitor includes three installed basic Profiles:

  • service_ping - Monitors a remote server via ICMP

  • snmp_network - Monitors any network device using SNMP

  • ssh_unix - Monitors a UNIX server

Additional supported Profiles are distributed with GroundWork Monitor. These XML Profile files are loaded and stored in the /usr/local/groundwork/profiles folder upon a GroundWork Monitor installation and need to be imported. This process enables users to import only those files that are needed.

The Profile Importer tool is used to import, update modifications, or import customized Profiles into the Monarch database. You can also select to remove unwanted Profiles. The Importer will list the available Profiles that are in the /usr/local/groundwork/profiles folder, import the selected XML files, and automatically search and import the corresponding Performance Configuration companion file (if there is one). Any associated Plugins will need to be copied to the GroundWork Server.

Importing Service Profiles

Show/Hide

Figure: Importing Profiles

  1. Select Profiles from the Configuration application.

  2. Select Profile Importer.

  3. In the Profile Importer screen you will see a list of all eligible files that are currently in the Import Folder directory /usr/local/groundwork/profiles.

  4. Select the box next to each of the files you wish to import. Service Profiles are displayed in the format service-profile-serviceprofilename.xml and Services are displayed in the format service-servicename.xml. The importer will import the Service Profile or Service and any associated perfconfig entries.

  5. Select Overwrite Existing Objects (located at the bottom of the screen) if you wish to overwrite existing objects and replace all matching time periods, commands, templates, services and profiles.

  6. Next, select Import.

  7. An Import Status screen will be displayed.

Uploading from Desktop

Show Hide

Figure: Uploading Profiles

  1. Select Profiles from the Configuration application.

  2. Select Profile Importer.

  3. In the Profile Importer screen you will see a list of all eligible files that are currently in the Import Folder directory /usr/local/groundwork/profiles.

  4. Scroll down to the bottom of the screen to browse for and upload a file from your desktop.

  5. Continue with the steps for Importing Service Profiles or Services above.

Removing Files from the Profiles Folder

Show/Hide

Figure: Removing Profiles

  1. Select Profiles from the Configuration application.

  2. Select Profile Importer.

  3. In the Profile Importer screen you will see a list of all eligible files that are currently in the Import Folder directory /usr/local/groundwork/profiles.

  4. Select the box next to each of the files you wish to remove.

  5. Scroll down to the bottom of the screen and select Remove.

Exporting Profiles

Profile files can be generated from the Export option found with the Host (Configuration>Hosts>Host) and Service (Configuration>Services>Service) profile pages. Files will be exported to the /tmp directory.
