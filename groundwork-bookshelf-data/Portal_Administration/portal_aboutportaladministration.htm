Portal Administration

About Portal Administration

This section of Bookshelf, PORTAL ADMINISTRATION, covers the GroundWork Monitor Administration feature. The Administration page is only available to user accounts assigned the Administrators role (GWAdmin). See Administration How To's for reference on common administrator management tasks. And for additional reference materials regarding JBoss Portal Management see Open Source Reference>JBoss Portal.

Note: Bolded titles below link to associated pages. Pages are also accessible using the Bookshelf's left side navigation.

Figure: Portal Administration

Administration has 5 sub pages;

  • Portal Management - Allows an administrator to manage portal objects, portlet instances, and portlet definitions.

  • User Management - The sub tabs under the User Management portlet include User Management and Role Management. The User Management page allows an administrator to manage system Users (e.g. admin, operator, user). Role Management is used to manage system Roles, (e.g. GWAdmin, GWOperator, GWUser). Restricted access to the Dashboards and Host Group and Service Group role restrictions are also set within Role Management.

  • Foundation Management - Foundation is the data integration framework. Here an administrator can manage Foundation configuration, application types, properties, host groups, performance data, and consolidation criteria.

  • Views - This tab is used for the Views web configuration interface. You can use it to configure views and/or build your own Views maps using your browser.

  • GroundWork License - Location for entry of the GroundWork Monitor license key for new installations.
