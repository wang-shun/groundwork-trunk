Portal Administration

Administration How To's

This page offers various Portal Administration How To's categorized under User Management, and Portal Management.

Note: The Administration option is only available to user accounts assigned the Administrators role (GWAdmin).

User Management

  • How To: Create a user account

    [click the how to link again to close this window]

    System user accounts are used to log in to GroundWork Monitor. Users are associated with roles which determine system access privileges. Follow the steps below to create a new user account and assign established role(s).

    Note: GroundWork Monitor does not prevent multiple users from simultaneously modifying the same data element such as Configuration, Reports, Hosts, Host Groups, and Services.

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. You will be on the first tab option; User Management.

    3.    Select the Create new user account link.

    4.    Enter the account properties as described in the table below and select Submit.

    5.    Next, enter a check in the corresponding box for each desired user role, then select Submit.

    6.    Select Submit one more time to confirm the account settings.

    Figure: User Management

    Username

    Name of the user to access GroundWork Monitor, [e.g. dave].

    E-Mail

    Email for the GroundWork Monitor user, [e.g. dave@gwos.com].

    Password

    User password. Must be a minimum of 6 characters, [e.g. davedave].

    Confirm Password

    Confirmation of the Password entered above, [e.g. davedave].

     

  • How To: Change a users password

    [click the how to link again to close this window]

    Passwords are part of a user's account profile. The profile page allows Administrators to change a users password, email address, add general preferences, personal information, instant message identities, and additional information. Administrators can edit all user profiles and Users and Operators can edit their own profile with the My Preferences link in the upper right corner of the portals page. Follow the steps below as an Administrator to change a users password.

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. You will be on the first tab option; User Management.

    3.    Select the Search users link.

    4.    Next, search for a user if necessary, then select the Edit user action on the line of the user account to be changed.

    5.    On the profile page, select the Change password link.

    6.    Enter a new Password and Confirm the password. Note: Passwords must be a minimum of 6 characters, [e.g. davedave].

    7.    Click Submit.

    Figure: User Management

     

  • How To: Disable a user account

    [click the how to link again to close this window]

    Registered user accounts can be disabled and re-enabled. This feature is useful if the user account needs to be temporarily inactive versus removed from the system. Follow the steps below to disable a user account.

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. You will be on the first tab option; User Management.

    3.    Select the Search users link. A list of established system users will be displayed in ascending order. The accounts currently enabled are marked with a check in the Enabled column.

    4.    To disable an account simply click the Disable action on the line of the user account to be disabled. This will uncheck the Enabled box. To re-enable, select the Enable action.

    Figure: User Management

     

  • How To: Remove a user account

    [click the how to link again to close this window]

    Deleting an account completely removes it from the GroundWork Monitor system. If you need an account to be temporarily inactive you can disable an account versus deleting the account. Follow the steps below to remove a user account.

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. You will be on the first tab option; User Management.

    3.    Select the Search users link. A list of established system users will be displayed in ascending order.

    4.    Select the Delete action on the line of the user account to be deleted.

    Figure: User Management

     

  • How To: Edit a users profile

    [click the how to link again to close this window]

    The account profile page allows Administrators to change a users email address, password, add general preferences, personal information, instant message identities, and additional information. Administrators can edit all user profiles and Users and Operators can edit their own profile with the My Preferences link in the upper right corner of the portals page. Follow the steps below as an Administrator to edit any user profile.

    5.    Make any edits and click Submit to save.

Role Management

  • How To: Create a new role

    [click the how to link again to close this window]

    Roles are used to grant different permission levels to different portlets, pages, or portal instances. A User account can be assigned several roles. The default system roles include Administrators (GWAdmin), Operators (GWOperator), and Users (GWUser) where GWAdmin is given administration permissions.

    Additionally, the roles MSP role (msp-sample) and Read Only Dashboard (ro-dashboard) are included in a clean installation. The msp-sample role does not have any default members assigned and has the host group Linux Servers as the visible and default host group. The ro-dashboard role does not have any default members assigned and there are no default restrictions for host group and service group visibility, although this role restrictions have be set to disable links to the Status application from all dashboards.

    Roles are created and assigned to users in the User Management portlet. They are associated with the portal objects using the Portal Management portlet. Follow the steps below to create a new role and then go to the how to: How do I assign user roles?

    By default new roles are assigned the portal permissions of a User role, (no administration access). To set role permissions see the how to: How do I assign Portal Objects to a role?

     

    Create a Role

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. Select the second tab option; Role Management.

    3.    Select the Create role link.

    4.    On the next page, enter a role in the Role box which will be a short name used for reference (e.g. Exec), and a Display name (e.g. Executives) for displaying to the user.

    5.    Optionally set Dashboard and Host Group and Service Group restrictions. For details see Role Restrictions. Note: Below are some rules that an administrator must be aware when creating a new system role:

    • If a user is member of multiple roles unrestricted access takes precedence.

    • If a user is member of multiple roles which is a mix of restricted (MSP view) and unrestricted access a user will inherit the unrestricted view (e.g. all host groups and service groups are visible to the user).

    • If a user is a member of multiple roles, the user can see a union of the host groups and service groups defined for the roles.

    • If a user is member of multiple roles that enable and disable links into the Status application, the user will have the links to the Status application enabled.

    • In all cases unrestricted access takes precedent over restricted access.

    6.    Click Submit. You will see the new role listed in the Role Management screen. Next you will need to assign this role to a user. See How To: Assign user roles?

    Figure: Create a New Role

    Figure: Role Management

    [click the how to link again to close this window]

    Roles are assigned to user accounts to determine system access privileges. Follow the steps below to add/remove a role from an existing user account.   • How To: Disable links to the Status application for a specific role

    [click the how to link again to close this window]

    Dashboards are the preferred way to show the current status of the system or sub systems for executives. In some cases it is desirable to disable the user's ability to drill-down from dashboard components, e.g. Event Console portlet, Service List portlet, and the Seurat view to the Status application. Selecting the box next to Disable links to Status viewer from all dashboards in the Edit Role page disables a roles associated users from being able to drill-down to status information from all dashboards.

     

    Disable links Status from all dashboards

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. Select the second tab option; Role Management.

    3.    Select the Edit role link next to the role you want to edit.

    4.    On the next page, click Disable links to status viewer from all dashboards box. When this box is checked all dashboard links will be disabled for users of this role. For details see Role Restrictions.

    5.    Click Submit.

    Figure: Disable links to Status from all dashboards

    Figure: Viewed in Dashboards>Monitoring

     

  • How To: Restrict host groups and service groups visibility for a specific role

    [click the how to link again to close this window]

    By default all roles and its associated users can view the entire infrastructure of available host groups and service groups. However, using role administration it is possible to restrict a role's visibility of host and service groups to show only a subset of a network's infrastructure. This means that only the selected host groups and service groups for a role will be visible within the Status, Event Console, Dashboards (including My GroundWork), and the Advanced Reports applications. Note that users associated with the GWAdmin role can see the entire infrastructure in all cases.

    Within the selected host and service groups to be visible, a default host group and a default service group can be selected. These selections result in default page that is displayed in the Status and Dashboards applications versus the display of the Entire Network view.

     

    Setting Host and Service Group Restrictions

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. Select the second tab option; Role Management.

    3.    Select the Edit role link next to the role you want to edit.

    4.    On the next page, select the Restriction Type as No Restrictions to show all host and service groups for users of this role. Alternatively you can select the radio button for Visible host groups and service groups. You will then need to check off each host group and service group you would like to have visible, all others will not be visible within the Status, Event Console, Dashboards (including My GroundWork), and the Advanced Reports applications. Make sure you select a default host group and service group from the selected visible groups to be the default group that will be displayed.  For details see Role Restrictions. Note: Below are some rules that an administrator must be aware when creating a new system role:

    • If a user is member of multiple roles unrestricted access takes precedence.

    • If a user is member of multiple roles which is a mix of restricted (MSP view) and unrestricted access a user will inherit the unrestricted view (e.g. all host groups and service groups are visible to the user).

    • If a user is a member of multiple roles, the user can see a union of the host groups and service groups defined for the roles.

    • If a user is member of multiple roles that enable and disable links into the Status application, the user will have the links to the Status application enabled.

    • In all cases unrestricted access takes precedent over restricted access.

    5.    Click Submit.

    Figure: Select visible host groups and service groups and set defaults

    Figure: View restricted to CA HG1 and CA HG2 host gorups for user dave of role Exec

     

  • How To: Enable the Actions portlet for a custom Role

    [click the how to link again to close this window]

    By default, the Actions portlet is enabled for use in the Status application for the roles GWAdmin and GWOperator only. However, an administrator can enable the Actions portlet for any custom role. Roles which have the Actions portlet enabled, are allowed to apply Actions to host and service groups, hosts and services in the Status application.

     

    Enabling Actions Portlet

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration, then select the User Management portlet. Select the second tab option; Role Management.

    3.    Select the Create role link (or you may edit an existing role), and check the Enable Actions Portlet box.

    4.    Click Submit.

    5.    You will need to log out and back in to GroundWork Monitor for the setting to take effect.

    Figure: Enabling Actions Portlet

    Figure: Actions Portlet used in the Status application

     

Portal Management

  • How To: Set a default portal page

    [click the how to link again to close this window]

    By default, when a user logs in they are forwarded to the default page of the default portal. The default Portal is groundwork-monitor and the default Page is dashboard. Follow the steps below to set your own default page.

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration. You will be in the Portal Management portlet.

    3.    Select the Portal link; groundwork-monitor.

    4.    Select the Make Default action next to the page to set as the default. This page will be displayed upon the next system login. The figure below show dashboard as the Default page.

    Figure: Portal Management

     

  • How To: Create a new page

    [click the how to link again to close this window]

    Any web page can be a JBoss Portal page and be included in the GroundWork Monitor user interface. Some examples of applications you might include as a page:

    • Help Desk application

    • Network centric monitoring solution such as Cisco works

    • Hardware management consoles such as HP Systems Insight Manager, Dell OpenManage Systems Management, APC, or any other device management console such as SAN storage.

    • Corporate phone directory (may be tied in through a Help Desk)

    • Vendor maintenance portals

    • ISP status pages

    • MRTG network device graphing solution

    • Cacti network graphing solution

    Follow the steps below to create a new page. In our example we'll create a Page using the Web Page Portlet Instance, and we'll use Google as the Source URL. Note: To be able to edit other Portlet Instances in a Portal Page, select the RenderSet in the Page Theme Properties to dashboardRenderer.

    1.    Log into GroundWork Monitor as an Administrator.

    2.    Select Administration. You will be in the Portal Management portlet.

    3.    Select the Portal; groundwork-monitor.

    4.    Enter a new page name (e.g. Google) and select Create page.

    5.    Select the Page Layout action next to the new page name (e.g. Google).

    6.    Select the WebPageInstance Portlet Instance.

    7.    The Instance name will be displayed at the top of the page and highlighted, select Add under Page Layout to add the Portlet Instance.

    8.    Select your new Page to test. You can update the Source URL by selecting the Edit icon.

    Figure: Portal Management
