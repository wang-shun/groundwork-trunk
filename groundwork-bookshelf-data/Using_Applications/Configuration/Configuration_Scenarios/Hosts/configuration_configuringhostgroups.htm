Configuring Host Groups

A Host Group definition is used to group one or more hosts together for the purposes of simplifying notifications. When a host goes down, becomes unreachable, or recovers, Nagios will find which host group(s) the host is a member of, get the contact group for each of those host groups, and notify all Contacts associated with those contact groups. Host groups allow for flexibility in determining who gets paged for what kind of problems.

Each host that you define must be a member of at least one host group - even if it is the only host in that group. Hosts can be in more than one host group.

Defining a Host Group

Show/Hide

Figure: Host Group Properties

  1. Select Hosts from the Configuration page.

  2. Expand the Host Groups drop-down menu.

  3. Select New.

  4. In the Host Group Properties screen, enter a host group Name which should be unique and not the same as a contact group. Enter an Alias.

  5. Next, optionally enter Notes. This field is intended to describe common features of the host group (e.g. the business unit, the geographical location,or the team responsible for maintenance). This information is shown on the host group detail page within the Status application.

  6. In this screen select the hosts from the right side list to be included in this host group.

  7. Next, select the Contact Groups that should be notified whenever there are problems (or recoveries) with any of the hosts in this host group.

  8. We will by pass the optional host and service escalation ID directives for now.

  9. Select Add to save the new host group.

After setting up host groups you are ready to setup Notifications and Escalations.
