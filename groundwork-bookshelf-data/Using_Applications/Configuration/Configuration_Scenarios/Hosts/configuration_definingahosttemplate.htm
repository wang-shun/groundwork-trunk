Configuration Scenarios

Creating New Hosts

Hosts are used to define a physical server, workstation, device, etc. that resides on your network. Host Templates store common object properties that are used to define multiple hosts. Templates are used to reduce the number of repetitive entries when defining objects. For example, when defining a new host you would first create a host template with common properties. Then, to define a specific host, you apply the properties using a host template. Note that GroundWork Monitor does provide a generic-host template.

Let's first define a new host template, and then use the New Host Wizard to create a new host using a template. Select the screenshot icon below to view the host template page.

Defining a Host Template

Show/Hide

Figure: Defining a Host Template

  1. Select Hosts from the Configuration page.

  2. Select Host Templates.

  3. Select New.

  4. In the Host Template Properties screen, enter a Name (e.g. generic-host) for the new host template.

  5. Continue by entering a check for selected template properties. By selecting the ? question mark next to each field you can view a description of the directive which also indicates if it is required or optional.

  6. Host templates also enable you to define and manage optional custom object variables.

  1. When finished defining template properties and any custom object variables, select Save.

  2. Now that we have a new host template, you can use the New Host Wizard to create a new host using this template. You may also select the home icon above to return to the list of configuration scenarios.
