Configuration Scenarios

Creating a New Host using the New Host Wizard

When creating a new host there are many objects that can be configured into a host definition. However after providing a Name, Alias, and IP Address, a Host Template is the only required object, all others are optional. Below, each of the five host property screens are described in detail. To view each page, select the screenshot icon next to each title.

Defining a New Host

  1. Select Hosts from the Configuration page.

  2. Select Host wizard which will display the first screen, Host Vitals.

Host Vitals

Show/Hide

Figure: New Host Wizard

  1. In this screen, enter a Name for the new host (e.g. Printer_Scanner_Fax).

  2. Enter an Alias as a name to identify the host (e.g. Savin).

  3. Enter an IP address for the new host (e.g. 172.28.113.102). The Host Profile option applies a set of pre-defined services to a host. We will cover Host Profiles in another section.

  4. Optionally enter Notes. This field is intended to hold a free-text description of this item. It is displayed in the Status application, and can be included in notifications via the $HOSTNOTES$ macro. It is commonly used to provide additional information about the item such as the team in charge of maintenance, the asset tag, or other identifying data.

  5. Optionally select a Host Profile which can be used to aid the design and management of hosts.

  6. Select Next which will display the Host Properties 1 screen.

Host Properties 1

Show/Hide

Figure: Host Properties

  1. Here we can select our previously defined Host Template, default-host which will apply all of the directive settings from the template to our new host. Parents are typically routers, switches, firewalls, etc. that lie between the monitoring host and a remote host. A host can be assigned to one or more parent hosts. We will visit the Parents option in another section.

  2. Select Next to go to the Host Properties 2 screen.

Host Properties 2

Show/Hide

Figure: Host Properties

Host Groups are an arbitrary collection of hosts into named sets. The usage of host groups simplifies access control, drawing layering, status displays, notifications, scheduling maintenance, multi-server commands, and reports. Here you can add host groups for this host, in other words this host will be a member within the selected host groups. You will get a warning message during a Pre Flight Test or Commit if a host is not a member of any host group.

  1. Select and add a host group for this host.

In the screenshot example above, we selected the host group Linux Servers to include the host Printer_Scanner_Fax. Extended Host Info, 2d and 3d Status Map Coords, and Escalations are covered in other scenarios.

  1. Select Next to go to the Select Services screen.

Select Services

Show/Hide

Figure: Select Services

At this point there are no services selected for our example Host Printer_Scanner_Fax. If we had selected a profile from the first screen you would see associated services listed. However, you can still select a profile or select individual services to apply to this host.

  1. Let's select the profile snmp-network, and click Add Profile. A list of all services will be displayed that are now associated with the new host. Here you can also modify and discard any services. Additionally, you can add individual services from the list.

  2. Select Next to go to the last screen.

Save as Profile

Show/Hide

Figure: Save as Profile

  1. You can select Save as Profile to save the new host as a profile that can be used in the creation of other hosts, or just select Continue.

  2. At this point you may want to go ahead and create a few more new hosts. If you need to change any of the definitions, from the Configuration page select Hosts>Hosts>HostGroup>HostName>Detail. You will be in the Manage Host screen.

Select the home icon above to return to the list of configuration scenarios.
