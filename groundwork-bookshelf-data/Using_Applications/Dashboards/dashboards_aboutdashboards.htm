Dashboards provide status overviews of monitoring data and centralizes status and system information in a single convenient location. Dashboards are AJAX-powered and display real-time and dynamically updated monitoring status providing system users with quick answers to questions about overall network health, alerts regarding issues or problems, and information to help make decisions that impact the business. Dashboards incorporate critical data from available portlet instances and display this information on a web page where you can navigate and drill-down into specific host and service information.

In this section of Bookshelf we'll review the GroundWork Monitor Dashboards page which displays shared system dashboards viewable by all system users based on their role attributes. We'll also cover the My GroundWork page which enables a user to create dashboards with personal views of monitoring information that are private to each user account. Refer to Dashboards How To's for steps on how to create, edit, and delete shared and personal dashboards.

Shared Dashboards

GroundWork Monitor's standard system dashboards can be displayed by selecting Dashboards. The default dashboards include; Summary, Monitoring, Enterprise View, Trouble View, and Webmetrics. These shared dashboards can be viewed by all system users and preferences for shared dashboards can be set by the individual users.

  • Show/Hide Dashboards>Summary - This dashboard displays a Quick Navigation portlet outlining various components of the GroundWork system. Links can be selected under each component for direct access.

    Figure: Dashboards Summary

  • Additionally, this dashboard shows the NetworkServicePlugin portlet instance is used in this dashboard to provide general information regarding Notifications and a GroundWork Installation Report which provides the installation name and date, and component versions.

  • Show/Hide Dashboards>Monitoring - This dashboard displays This dashboard shows the Monitoring Performance portlet which displays statistics for the entire monitoring system in a series of gauges. The Service Latency gauge measures service latency in seconds, Service Execution measures the number of seconds for check execution, the Service Checks gauge measures the number of checks performed per minute. Also included are the portlets; Recent State Changes, and Performance Measurement.

    Figure: Dashboards Monitoring

  • Show/Hide Dashboards>Enterprise View - This dashboard displays the portlets; Seurat View (Host Group Snapshot: Entire Network), Host Group Status, and Service Group Status. The Seurat View portlet can be sorted using the drop-down Host Name. Blinking icons in this view represent any host or service that has not been acknowledged.

    Figure: Dashboards Enterprise View

  • Show/Hide Trouble View - This dashboard displays the portlets; Events: Entire Network, Host Status: Entire Network, Service Status: Entire Network, Troubled Hosts: Entire Network, and Troubled Services: Entire Network.

    Figure: Dashboards Trouble View

  • Show/Hide Webmetrics - This dashboard displays the Webmetrics Integration portlet which offers web performance management monitoring.

    Figure: Dashboards Webmetrics

Additional shared dashboards can be created by an administrator. Creating shared dashboards allows access to a particular subset of the infrastructure to specific users. It is possible to set up user roles to have access to individual pages or sub-pages, and to control exactly how much of the monitored infrastructure individual users can see and interact with.

Additionally, an administrator can create a Portal Page with specific Portlet Instances to be viewed by all users of a role. For example; the My GroundWork application enables users to create and personalize a monitoring view as a dashboard for individual use, whereas Portal Pages located in the top navigation (e.g. Event Console, Status, Reports) can be created and have global access based on user role permissions. See How To: Create a new page in the Portal Administration section of Bookshelf.

Personal Dashboards

The My GroundWork page allows users to personally configure and view their own dashboards. For example, If you login with the username user and create a dashboard, only the username user can view the dashboards created. Personal Dashboard portlets can be edited using the Edit icon to specify for example, the host to be displayed. See Using the Personal Dashboard Editor section below. Additionally, for how to reference on common Dashboard configuration tasks see Dashboard How To's

About Users and Roles

Personal dashboards are configured and viewable by username. For example, a personal dashboards created by username user can only be viewed by username user.

Additionally, it is important to remember when configuring dashboards that roles are associated with user accounts and roles limit the accessibility of specific portal pages and portlet instances. For example, the logged in username user is associated with the role GWUser which can not access the portal page Event Console (see table below). So, for dashboards this also applies to specific portlet instances. For example, all Event Console portlet instances will be unavailable to users associated with the roles; GWUser, msp-sample, and ro-dashboard when configuring a dashboard. And the Actions portlet used to execute commands in several portal pages is limited to GWAdmin, GWOperator, and the msp-sample roles.

  • The GWAdmin role (user account admin) has access to all portal pages and their portlets. Administrators can configure dashboards, have access to view all portlet instances in a dashboard, and maintain command execute permissions.

  • The GWOperator role (user account operator) has access to the portal pages Dashboards, My GroundWork, Event Console, Status, and Resources and their portlets. Operators can view and execute commands from the Actions portlet.

  • The GWUser role (user account user) can configure dashboards however the actual dashboard portlet access is limited; no viewing access to the Event Console, Views, Reports, Auto Discovery, Configuration, Administration, and Nagios portlets - and no access to the Actions portlet (command execution privileges).

  • The msp-sample role does not have any default members assigned and has the host group Linux Servers as the visible and default host group. This role is conveniently setup for a Multiple Service Provider.

  • The ro-dashboard) role is also included in a clean installation. This role does not have any default members assigned and there are no default restrictions for host group and service group visibility. Additionally, this role's restrictions have be set to disable links to the Status application from all dashboards.

Using the Personal Dashboard Editor

Show/Hide You can configure dashboards by selecting the My GroundWork page and then selecting Configure. Selecting this option opens the Personal Dashboard Editor which enables each user to create personal dashboards associated with their login. The editor page is used to manage the content and appearance of your dashboard pages. You can control the page titles, column layouts, and add, order, or remove portlets.

Figure: Configure Dashboard

The five main sections of the editor page include;

  1. Currently editing page which indicates the current dashboard being edited

  2. Create a New Page used to name your new dashboard page

  3. Theme and Layout to set a pre-defined layout or column format for the new page.

  4. Content Definition - The location where all available portlet instances are selected.

  5. Page Layout - where selected portlets are positioned in the dashboard page

GroundWork Portlets Available for use in Dashboards and My GroundWork

This section describes each GroundWork portlet instance that a user can place in the Dashboards or My GroundWork portal pages.

GroundWork runs under the JBOSS portal, and the user interface is composed of portlet applications. Various GroundWork portlets are not available for use in dashboards since many of them depend on other portlet associations or configuration data in order to work. GroundWork portlets available for use in Dashboards and My Groundwork are listed and described in detail below. This list of portlets is controlled by the file /usr/local/groundwork/config/mygroundwork.xml. If you add portlets from an external source or of your own design to the system, you will need to add them to this file to make them visible.

Dashboard Portlets

In the descriptions below you will notice that in the dashboard view mode (Dashboards or My GroundWork), many of the portlet instances can be setup to view specific objects by selecting the Edit icon which is located in the upper right corner of the displayed portlet instance. When editing preferences, various fields utilize an Auto Complete feature which provides a list of options as you type, for example when you start typing the host name localhost a list of similar spelled host names will appear. Additionally, when entering a or host or host group, service or service group that is not available (e.g. NO_SUCH_HOST), the portlet will display an information message indicating a message similar to No Host available with name [NO_SUCH_HOST]. In the case of host groups, if you specify blank (" ") host group name the portlet will display events for the entire network.

Show/Hide Event Portlet [Enterprise only] - This portlet is an embedded Event Console application which provides an event list for the entire network or for a selected host group.

The Event Portlet lets you apply event actions (e.g. Accept Log Message, Notify Log Message, Nagios Acknowledge), sort, and pause incoming events. This portlet is also viewable at the service group, host and service levels in the Status application. By default this portlet displays a list of events for the entire network or for a selected host group.

Figure: Event Portlet

This portlet instance includes the following editing preferences:

  • Host Group Name: You can enter the name of any available host group and the portlet will display a list of events for the entered host group.

  • Events Per Page: This is the page size configuration option. By default 5 events per page will be displayed in the portlet.

  • Custom Portlet Title: By default the title of this portlet is Events: <host group name or entire network>. With this configuration option you can set your own customized title for this portlet.

Figure: Event Portlet - Editing Preferences

Show/Hide Performance Measurement - Provides RRD graphs for the various host services.

The Performance Measurement portlet displays graphs which are arranged in the same order of the Host Availability portlet and aligned also by their timelines. Time filters, for example; Today, Last 24 Hours, Last 30 Days, Custom Date-Time etc, can be selected as needed. The integrated performance graphs display time-series data such as; network bandwidth, CPU utilization, machine-room temperature, transaction response times, and server load averages. These graphs are viewable at the host and service levels. Note: Graphs may not be available for a service unless configured by a system administrator using the Configuration>Performance feature in GroundWork Monitor.

Figure: Performance Measurement Portlet

This portlet instance includes the following editing preferences:

  • Host Name: You can enter name of any system host and the portlet will display associated performance graphs for the entered host and selected host services.

  • Service Name: Enter a name of the selected host services and the portlet will display associated performance graphs for the entered host services.

  • Custom Portlet Title: By default the title of the portlet is Recent State Changes: <host name>. With this configuration option you can set your own customized title for this portlet.

  • Time: You can select and apply time filters (e.g. Today, Last 24/48 Hours, Last 5/7/30/90 Days; and a Custom Date-Time). After selecting and applying the filter the portlet will display RRD graphs for the selected timelines. For example, RRD graphs for the last 5 days will be displayed if the user selects :Last 5 Days as the time filter. This Time option can also be changed when viewing the portlet.

Figure: Performance Measurement Portlet - Editing Preferences
