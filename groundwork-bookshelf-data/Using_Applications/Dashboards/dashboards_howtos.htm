Personal Dashboards (My GroundWork)

  • How To: Create a personal dashboard

    [click the how to link again to close this window]

    The three main sections of the editor page includes Create a new page to name your new dashboard page; Theme and Layout to set a pre-defined layout or column format; and Content Definition where portlets are selected to be part of the new page. Follow the steps below to create a new dashboard.

    Note: The default theme is groundwork. Other themes are not supported at this time.

    1.    In GroundWork Monitor select My GroundWork and then Configure. You will then be in the Personal Dashboard Editor.

    2.    In the upper-left corner of the editor enter a page name for your dashboard and select Save.

    3.    Next, just below under Theme and Layout choose a dashboard layout (1 column, 3 column, or choose generic for a left-plus-center layout similar to the Status page), and click Select. You will see your selected layout format displayed on the right side of the screen under Page Layout.

    4.    Next, we'll define the dashboard content by adding Portlet Instances to the various sections of the dashboard layout. To do this first select an instance from the list under Portlet instance associated to this window. After selecting an instance it will be displayed at the top of the list and highlighted in blue, select Add within one of the regions of your Page Layout.

    5.    Continue selecting instances for your dashboard. When you are finished select the new Dashboard name from the secondary portal navigation. Your new personal dashboard will be displayed.

    6.    Many of the portlet instances can be configured to display specific objects (e.g. show only service in a non-OK state) by selecting the Edit icon in the upper-right corner of each instance. See example screenshots below for setting preferences for the Events, Host List, and Service List portlets.

    7.    You can dynamically rearrange portlets in My GroundWork by dragging the title bar of a portlet to a new location.

  • How To: Edit a personal dashboard

    [click the how to link again to close this window]

    Any of your personal dashboards can be edited. The three main sections of the editor page includes Create a new page to name your new dashboard page; Theme and Layout to set a pre-defined layout or column format; and Content Definition where portlets are selected to be part of the new page. Follow the steps below to edit a dashboard.

    Note: The default theme is groundwork. Other themes are not supported at this time.

    1.    Select My GroundWork and then Configure. You will then be in the Personal Dashboard Editor.

    2.    At the center of the screen select the page you would like to edit. You should then see the title change to Currently editing page [pagename].

    3.    Make any necessary changes and your dashboard is ready to be viewed. Remember many of the portlet instances can be configured to displayed specific objects by selecting the Edit icon in the upper-right corner of each instance.

    Figure: Editing Dashboards

     

  • How To: Delete a personal dashboard

    [click the how to link again to close this window]

    You may delete dashboard pages using the Personal Dashboard Editor. You cannot delete the default dashboard page. Deleted pages are NOT recoverable. This change is permanent.

    1.    Select My GroundWork, Configure. You will then be in the Personal Dashboard Editor.

    2.    At the center of the screen select the page you would like to remove. You should then see the title change to Currently editing page [pagename].

    3.    Next, scroll down to the bottom of the screen and select the Delete this Page button. The dashboard page will be deleted and is not recoverable.

    Figure: Personal Dashboard Editor

     

Shared Dashboards (Dashboards)

[for Administrators]

  • How To: Create a shared dashboard

    When creating shared dashboards it’s a good idea to plan out the dashboard pages you wish to add and the roles/users you wish to assign access to before you start. This will allow you to quickly and easily set up several pages in a consistent manner.

    The general workflow is to create a dashboard page, add portlets to the new dashboard page, set portlet preferences, and then define dashboard access.

    Before getting started you should create any new Role(s) and User(s) that will apply to the new shared dashboard and then continue with Create a Dashboard Page below. Throughout this How To we will use the following in our example:

    • Role: R1 Users: Sally, Jim

    • Dashboard Page Name: Region1

    • Theme Layout: 1column

    • Dashboard Page Portlets: Host Group Health, Host List, Service List

    • Portlet Preferences:

    Host Group Health: Host Group Name - R1, Custom Portlet Title: Host Group R1: Availability

    Host List: Host Group Name - R1, Custom Portlet Title - Host Group R1: Hosts

    Service List: Entity - Host Group, Name - R1, Check Show service that are not OK, Custom Portlet Title - Host Group R1: Troubled Services

    Figure: Shared Dashboard Example

    Create a Dashboard Page

    1. Log into GroundWork Monitor as user admin.
    2. Select the Administration application.
    3. Under the Portal section, select groundwork-monitor. Note: Do not select the template portal as it is used only if you are developing new functionality, or you have been asked to do so by GroundWork support.
    4. Under the Page section, select dashboard.
    5. In the Create a page named: box type the name of your new dashboard. Select Create page. You will see the new page name displayed.
    6. Select the Theme action option on the corresponding line of your new page, then select the Layout drop-down box and choose a page layout (e.g. 1 column, 3 column, or choose generic for a left-plus-center layout similar to the Status page).
    7. Select the RenderSet dashboardRenderer , (later you will be instructed to change this back to default).
    8. Click the Update button.

    Add Portlets to the Dashboard Page

    Next, select Switch to wizard mode located in the lower right corner. This brings up a specific workflow which you MUST use when placing portlets on created pages. Do not attempt to place portlets on pages ad-hoc. It will likely result in unusable dashboards. Follow steps 1-5 as outlined below.

    Note: You will need to do this for each portlet that you want added to the dashboard page:

    STEP 1 - Select Portlet

    Select from the available dashboard portlets.

    Status Viewer -

    • Host Availability Instance - Displays color-coded status for a selected Host.

    • Host Group Health Instance - Displays the status for the selected host group’s host and service availability.

    • Host Group Summary Instance - Shows monitoring statistics at the entire network level with drill-down capability for viewing detailed data.

    • Host Health Instance - Provides a quick status and information for the selected Host.

    • Host Information Instance - Shows detailed status and check interactive information for selected Host. Custom links can be configured.

    • Host List Portlet Instance - Provides a listing and status of the Host Group's Hosts.

    • Host Summary Instance - Shows monitoring statistics at the Entire Network and Host Group levels with drill-down capability for viewing detailed data.

    • Service Availability Instance - Provides color-coded status for an indicated Host Service.

    • Service Group Health Instance - Provides a quick status for the indicated Service Group’s service availability.

    • Service Group Summary Instance - Shows monitoring statistics at the Entire Network level with drill-down capability for viewing detailed data.

    • Service Health Instance - Provides quick status and information for the selected Service.

    • Service Information Instance - Shows detailed status and check information for the selected Service.

    • Service List Instance - Provides a list of Services the their status for the Entire Network, or an indicated Host, Service Group, or Host Group.

    • Service Summary Instance - Shows monitoring statistics at the Entire Network, Host Group, Service Group and Host levels with drill-down capability for viewing detailed data.

    Event Console -

    • Events Portlet Instance - This portlet is an embedded Event Console application which provides an event list for the Entire Network or for a selected Host Group, Service Group, Host, or Service.

    Monitoring Performance -

    • Monitoring Performance Instance - Displays statistics for the entire monitoring system in a series of gauges.

    Notification and Installation

    • Network Service Plugin - Provides GroundWork notifications and installation information to the user. "Notification Service" needs to be enabled.

    Performance Graphs -

    • Performance Measurement Instance - Provides RRD graphs for the entered Host Service.

    URL -

    • Web Page Instance - Embed a web page within a Dashboard.

    Seurat View

    • Seurat Instance - Offers an overall view of a Host Groups Hosts status sortable by Host name, status, last state change, along with the ability to drill-down directly into a Hosts detail in the Status application.

     

    STEP 2 - Select Portal

    In this step, always select the portal groundwork-portal.

     

    STEP 3 - Select Page

    Here we will select the page to place the selected portlet. This is the sub page of the dashboard page you created in the previous steps. Select Sub-Pages link on the corresponding line for dashboard. Select your new dashboard page.

     

     

    STEP 4 - Select a Region

    Next, select a listed region. If your layout contains more than one region, for example generic layouts have a left and a center region, you must chose which to place the portlet in.

     

    STEP 5 - Confirm

    Validate your selection and select Confirm. Continue selecting instances for your dashboard.

    Set Portlet Preferences

    GroundWork has modified the JBoss portal to allow inheritance of portlet preferences settings from the admin user. Preferences for shared dashboards are set by an admin user and can then be set by the individual user as preferences are stored with the individual user in JBoss. This means that a user who accesses a dashboard will see their portlet preferences if they have been edited, or the admin user preferences or default settings for the portlet if they have not.

    1.    When you are finished adding portlets to your dashboard page, select your dashboard from the Dashboards page.

    2.    Many of the portlet instances can be configured to display specific objects (e.g. show only services in a non-OK state) by selecting the Edit icon in the upper-right corner of each instance.

    3.    See the example screenshot below for setting preferences for the Service List portlet.

    Note: When all portlet preferences are set, return to Administration>Portal Management, and drilldown to the dashboard page you created. Select Theme, and change the RenderSet back to default. This will lock the users out of the edit preferences screen.l

    Define Dashboard Access

    When creating shared dashboards you can define the access to a particular subset of the infrastructure to specific users. It is possible to set up user roles (e.g. R1) to have access to individual pages or sub-pages (e.g. Region1), and to control exactly how much of the monitored infrastructure individual users can see and interact with. Follow the steps below to allow only the users Sally and Jim who are members of the role R1 to access the new shared dashboard Region1.

    This process involves setting the security access for pages and portlet instances. First, you should change the default portal page in case you make a mistake in setting up the access for the dashboard pages. You should always shift the default portal page away from the pages you are setting access to, to ensure that you don’t have to reset the portal database and lose all your dashboards.

    Change Default Page

    1. Select the Administration page, and then Portal Management.
    2. Select the groundwork-monitor portal.
    3. Change the page Default to another portal page. We suggest you use Status since all users can normally access this portal. You can change the default back to Dashboards at the end of this entire process. In the example below we change the default from dashboard to status.

    Setting Dashboard Page Access

    Now set the access you want for the Dashboards page. This allows you to restrict which roles can see Dashboards, later we'll allow specific users (e.g. Sally, Jim) to view individual dashboards (e.g. Region1).

    1. Select the Administration page, and then Portal Management. Select the groundwork-monitor portal. Select the dashboard page, and select the Security icon for this page.
    2. Un-check all the checked boxes, and then check View Recursive and Personalize Recursive for the Administrators role. Also, choose which roles (e.g. R1...) that will get to see ANY dashboards at all and give them View permissions as in our example below. Click Update.

    Setting Dashboards Sub-Page Access

    Next, we'll set the access for a specific Dashboards Sub-Page. Here we choose which Sub-Pages (e.g. Region1) the Role (e.g. R1) will see.

    1. Select groundwork-monitor portal in the breadcrumbs section of the current Portal Objects page.
    2. Under Page, select dashboard.
    3. Under Page again, select Region1 (or your new dashboard).
    4. Under Manage [dashboard sub-page name] page select the Security property icon.
    5. Now we need to select View for the Roles (e.g. R1) we wish to give access to the dashboard (Region1 or your new dashboard). To allow any role's users to change the portlet preferences, also select Personalize. Note: The user preferences will supersede those selected by the admin user.

    Setting Dashboards Default Summary Page

    The Dashboards page has a default sub-page (set to the Summary page). There is no easy way to change this, so we suggest you either modify this page, or allow access to it for everyone who can access any dashboard.

    1. Select dashboard page in the breadcrumbs section of the current Portal Objects page.
    2. Under Page, select summary.
    3. Under Page again, select Region1 (or your new dashboard).
    4. Under Manage summary page select the Security property icon.
    5. Now we need to select View Recursive for the Roles (e.g. R1) we wish to give access to the dashboard (Region1 or your new dashboard). It's a good idea to give all roles access to this dashboard page.

    Now you are ready to test out your new shared dashboard. You should be able to login as sally or jim and see the dashboard Region1. If you login as admin you can also view the Region1 dashboard. But if you login as operator or user you will see that Dashboards is not even an option. If we would have selected the operators and users role in the dashboard page security permissions then you would be able to view Dashboards but not the Region1 dashboard which we gave only view permissions to the R1 role. Remember, operators and users role should be given access to the summary dashboard too.

    Setting up Top-Level Pages

    The Administrator can use the same procedure to create top-level pages in the portal. The process is nearly the same: Create the page, use Wizard mode to place the portlets, and then set the preferences. There is an additional step, however.

    Once the portlets are placed, you must set the render set to dashboardRenderer to enable editing of the portlet preferences.

    1. Select Administration and then Portal Management.
    2. Select the page you have created and select the Theme icon. One of the options is Layout, as we covered before, and another is RenderSet. Choose dashboardRenderer as the RenderSet.
    3. Now you can proceed to edit the portlet preferences. Once this is complete, you may change the render set back to default to restrict the ability to edit the portlet preferences.

    Example: Restricted page

    As an example, we will add a page to the top level called Restricted, and place a web page portlet on it that can point to an internal web application.

    Create a Page at the Top Level

    1. In Administration, Portal Management, select the portal groundwork-portal and type in Restricted in the Create a Page named: box.
    2. Select the Theme icon for the page you created. Select the Layout you want, for example 1column, and select dashboardRenderer for the RenderSet. Select Update.
    3. Switch to wizard mode, and go to the bottom of the list of portlets. Select the local.iframeportlet.WebPagePortlet portlet to place a web page portlet on the Restricted page. Then select portal groundwork-monitor, the page Restricted, and region center (in our case), and confirm.
    4. Next, go to the Restricted portal page. Select the edit icon, and set the URL preference to the address of the internal web application. Save these preferences.
    5. Next, return to Administration, and go to Advanced mode. Select the Theme icon for the Restricted page, and return the RenderSet to default. Update.
    6. Now you are ready to set Security for the Restricted page. For example, if you want only Administrators to see it, select the Security icon, and select View for the Administrators role.

  • How To: Delete a shared dashboard

    [click the how to link again to close this window]

    You may delete shared dashboard page as an admin user. Deleted pages are NOT recoverable. This change is permanent.

    1.    Select Administration and then Portal Management.

    2.    Select the Portal groundwork-monitor.

    3.    Select the Page dashboard.

    4.    Under the Page section, select the dashboard Page you wish to remove.

    5.    Under Manage [page name] section, select Delete. You will need to confirm the deletion. The dashboard page will be deleted and is not recoverable.
