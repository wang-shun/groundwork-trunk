Viewing Events

  • How To: Use filters to display specific events

    [click the how to link again to close this window]

    The Event Console provides viewing options for different monitoring data types. When entering the application the most recent open events (All Open Events) will be displayed in descending order (most recent event first) by the column titled Received By GW. There are 20 events listed per page with pagination controls displayed at the bottom of the screen. Specific events can be displayed by using defined System or Public filters. System filters are automatically generated for objects and groups in the system and Public filters are pre-defined. Follow the steps below to filter console events by system and public filters.

    1.    To select a System filter click on the symbol next to the group filter title (e.g. Applications, Host Groups) and then select a filter. The right side of the screen will update the event list with only those events matching the selected system filter. Public filters are used similarly.

    2.    From here you may want to apply search parameters to refine your list of events.

    Figure: Using Filters

     

  • How To: Search for specific events

    [click the how to link again to close this window]

    The Search Events feature enables filtering of events on top of the applied filter from the side panels. A user can indicate a device name and/or message content to be searched. Also, a date and time range can be set to search only those events matching an indicated preset time (e.g. Last 10 Minutes)  or a selected calendar date range. The figure below displays an All Open Events with a search criteria set as: Messages: SNMP problem. Follow the steps below to search for specific events.

    1.    Within the Event Console page, select a System or Public filter or you can search in the All Open Events default tab.

    2.    In the Search Events and Date-Time Range areas indicate the search parameters, (e.g.Device: 127).

    3.    Select the Search button. The events matching the search criteria will be displayed. Selecting Reset button will remove all previously set search criteria and returns to the filtered events.

    Note: Entering more than one parameter (e.g. Device, Message, and Data-Time Range) results in and logical AND type search where all parameters are included in the search results. Using more than one parameter will help narrow down the search.

    Figure: Searching in the Event Console

     

  • How To: Add a new tab

    [click the how to link again to close this window]

    The New tab feature enable multiple event lists to be displayed each with their own filtered data so a user can easily access and switch between multiple defined lists. Follow the step below to add a new tab.

    1.    In the Event Console page, select New tab. You will be presented with a new tab with the default filter of All Open Events.

    2.    Next, select a filter along the left pane for the events you wish to display (e.g. Critical). You can enter your own tab title using the Update Label option (e.g. CRITICAL). Next click the Update Label button.

    3.    To close a tab select the red X on the tab to close. Closing a tab also removes it. There must be more than one active tab in use before you can close a tab.

    Figure: Adding a New Tab

     

  • How To: Sort events

    [click the how to link again to close this window]

    Sorting and filtering functions within Event Console allow users to focus on events of given types, time periods, or sources. Follow the steps below to sort an event column.

    1.    In the Event Console page, click on one of the event column titles. The column will be sorted in descending order, indicated by a down arrow symbol. Click again to sort in ascending order, indicated by an up arrow. By default, events are sorted by the Received by GW column in descending order.

    Figure: Sorting Events

     

  • How To: Collapse and re-display console panels

    [click the how to link again to close this window]

    Collapsing panels in Event Console allows more event data to be displayed. Follow the steps below to collapse and re-display event screen panels.

    1.    To hide the panels, select the side filters pane arrow and the top search pane arrow.

    2.    Re-select the side filters pane arrow and the top search pane arrow to re-display the panels.

    Figure: View of Console before Collapsing

     

  • How To: Access Device details and Status from Event Console

    [click the how to link again to close this window]

    The Event Console application provides cross links to the corresponding Status pages. When a device name in the Device column is selected the device host page will be displayed in Status. If no host for that device exists (snmptrap) the link will be in-active (not clickable). If application type Nagios is selected any services in the Service column link to the Status service page.

    Additionally, a user can select the icon in the Device column to display device details.

Applying Actions

  • How To: Apply actions to events

    [click the how to link again to close this window]

    The Event Console enables users to select messages and apply various actions like Open Log Message, Close Log Message, Accept Log Message, and other application type specific actions on the selected message(s). Also, developers can configure additional actions; see the Developer Reference>Event Console>Configuring Actions for information on how to add additional event actions. Follow the steps below to apply an action to an event or group of events.

    1.    First you will need to select the event or events to apply an action to. To select a single or multiple events simply click the events you want to include. To deselect click the event again. Use the Select All button to select all events.

    2.    Once the events are selected, click the Actions button and select the appropriate listed action.

    3.    Once actions have been applied to the events the events will be relocated to the Operation Status folder. All Open Events will continue to display an entire listing of events.

    Note: Once a message is selected the system automatically pauses incoming events. Incoming events will be resumed upon a user selecting an action from the menu or by selecting the Resume Incoming Events button manually.

    Note: The list of actions displayed is driven by the message(s) that are selected. There are two sets of action types. Standard System Actions which include  OPEN, CLOSED, NOTIFIED, and ACCEPTED; and Application Actions which include Nagios Acknowledge, and Submit Passive Check. If the message(s) selected belong to a single application type, the action list will consist of Standard System Actions plus the specific Application Actions. If the message(s) selected belong to multiple application types then only Standard System Actions will be listed. Refer to the table below for action definition descriptions.
