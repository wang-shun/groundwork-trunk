Setting up Log File Reports as a Data Source

Configuring MySQL for Log File Reports

The installation of the MySQL driver is used for Log Reports and availability information (dashboard database).  The Report Designer must use the JDBC Data Source in order to access this information.

  1. If you have not yet setup your database permissions you will need to start by running the following command to grant permissions to the log report user. For example:

Login to the groundwork server and execute the command: mysql

At the mysql> prompt type the command which gives permission to access the logreports DB from any Host:

grant all privileges on logreports.* to logreports@'%' identified by 'gwrk';  

  1. In the Report Designer, select the Data menu option.

  2. Select New Data Source.

  3. Click on JDBC Data Source and enter a Data Source Name (e.g. Log File Reports Data Source) in the space provided and click Next.

  4. Click on Manage Drivers.

  5. Select Add and browse to the MyDocuments/GWCustomReports directory to select the mysql-connector-java-3.1.13.jar file, select OK.

  6. Next, select the Driver Class com.mysql.jdbc.Driver (3.1) from the drop-down list in the first box.
