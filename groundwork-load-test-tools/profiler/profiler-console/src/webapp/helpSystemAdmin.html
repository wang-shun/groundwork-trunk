Overview

The SystemAdmin Profiler requires first to specify the number of devices, number of hosts, number of services (we default the number of devices to be equal with the number of hosts). The tool automatically tells foundation to initiate in the system the required number of devices, hosts and services.

The user can specify a in precentage how many checks will up/down, how many events, and how many messages are consolidated.

The tool will run the required host/service checks, and at the end will publish following lists of messages and the latencies/message:
1. SystemAdminInit
2. SystemAdminToggleHostStatus
3. SystemAdminTolggleServiceStatus
4. SystemAdminLogMessage

Installation/Configuration/Build

The code is downloaded from
svn://serverurl//groundwork-professional/trunk/load-test-tools/profiler/profiler-console After downloading it, in the directory profiler-console run
maven database build deploy

Run

The build and deploy goals will create a war file which will pe copied under webapps directory of foundation. After starting foundation the Foundation Profiler Tool is accessed at the web address:
http://localhost:8080/groundwork-profiler/

Reports

The output will be a html report. At the same time in the GWProfileDB, the tables MessageBatches, Workloads, and WorkloadSessions are updated with the results.
