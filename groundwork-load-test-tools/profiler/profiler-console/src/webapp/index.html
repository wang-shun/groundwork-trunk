Overview

The Foundation Profiler Tool is based on a legacy code which did profiling on the initial message type used by Foundation. Due to this fact the initial profiling tool uses concept such Workloads, WorkloadMessages, Batches, and was designed to use a xml configuration by which the tool was told how many workoads to build, how many messages in a workload to bundle, how many batches to build, and how many times the execution should be repeated.

Due to the introduction of the new message type called SystemAdmin, which allows to build transactions, therefore to bundle messages together, the Foundation Profiler Tool can be executed with fewer paramaters.

Initially the Legacy Code required to edit a xml configuration file, to setup the various paramaters. The new tool is web based, with a html interface.

Build/Configuration

The tool is stored on svn at the address:
svn://serverurl//groundwork-professional/trunk/load-test-tools/profiler/profiler-console After downloading it, in the directory profiler-console run
maven database build deploy
The goal database is invoked only the first time, when the database is build, but also it can be invoked with goal to clean it.

The build and deploy goals will create a war file which will pe copied under webapps directory of foundation. After starting foundation the Foundation Profiler Tool is accessed at the web address:
http://localhost:8080/groundwork-profiler/
